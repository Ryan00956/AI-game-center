## ✨ 项目简介

**AI 游戏大厅** 是一个基于 Web 的 AI 社交推理游戏平台。你可以配置多个不同的大语言模型（LLM），为游戏中的每个 AI 玩家分配不同的模型，然后与它们一起玩经典的社交推理游戏。

### 🎯 核心亮点

- 🤖 **多模型对战** — 支持同时配置多个 AI 模型档案，为每个 AI 角色分配不同的模型
- 🐺 **狼人杀** — 完整的狼人杀游戏，包含预言家、女巫、猎人等经典角色，支持警长竞选机制
- 🕵️ **谁是卧底** — 经典词语推理游戏，包含平民、卧底、白板三种身份
- 🏰 **阿瓦隆** — 经典阵营博弈游戏，包含梅林、派西维尔、莫甘娜、刺客等角色
- 🎭 **沉浸式体验** — AI 玩家拥有独立记忆和策略，像真人一样自然对话
- 🌐 **多模型支持** — 兼容所有 OpenAI 格式的 API 接口

## 🕹️ 支持的游戏

### 🐺 狼人杀（警长版）

经典社交推理游戏。玩家在白天讨论中找出隐藏的狼人，或者作为狼人每晚偷偷猎杀村民。

| 特性 | 说明 |
|------|------|
| 👥 人数 | 6-11 人（含 1 名真人玩家） |
| 🎭 角色 | 狼人 × 2-4、预言家、女巫、猎人、村民 |
| 🌙 夜晚 | 狼人商议猎杀、预言家查验、女巫用药 |
| ☀️ 白天 | 警长决定发言顺序 → 讨论 → 投票淘汰 |
| 🎖️ 警长 | 第一天竞选警长，警长投票权重 1.5 票，死亡时传递警徽 |
| 💀 猎人 | 死亡时可带走一名玩家 |

### 🕵️ 谁是卧底

每位玩家拿到一个词语，大多数人拿到相同的词，卧底拿到相似但不同的词。通过描述词语来找出谁是卧底！

| 特性 | 说明 |
|------|------|
| 👥 人数 | 5-8 人（含 1 名真人玩家） |
| 🎭 身份 | 平民、卧底、白板（无词） |
| 🗣 玩法 | 每轮描述词语 → 投票淘汰 |
| 📦 词库 | 内置 30 组精选词语对 |

### 🏰 阿瓦隆

经典隐藏身份推理游戏。队长组队、全员投票、任务成败与最终刺杀交织在一起。

| 特性 | 说明 |
|------|------|
| 👥 人数 | 5-8 人（含 1 名真人玩家） |
| 🎭 角色 | 梅林、派西维尔、忠臣 vs 刺客、莫甘娜、爪牙 |
| ⚔️ 玩法 | 队长组队 → 投票通过/否决 → 执行任务 → 刺客刺杀 |
| 🗳 机制 | 5 轮任务制，连续 5 次否决则邪恶直接获胜 |

## 🚀 快速开始

### 前置要求

- [Node.js](https://nodejs.org/) (v18+)
- npm
- 至少一个 AI 模型的 API 密钥

### 安装与运行

```bash
# 1. 克隆项目
git clone https://github.com/Ryan00956/AI-game-center.git
cd AI-game-center

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

启动后在浏览器打开提示的地址（默认 `http://localhost:5173`）。

### 构建生产版本

```bash
npm run build
npm run preview
```

## 📖 使用指南

### 1. 配置 AI 模型

点击页面右上角的 **「模型配置」** 按钮：

1. 点击 **「添加新模型档案」**
2. 输入档案名称（如 "Gemini Flash"）
3. 填写 API 密钥
4. 选择 API 地址（默认 OpenAI 官方，也可用其他兼容接口）
5. 选择模型或输入自定义模型名称
6. 可以添加多个不同的模型档案！

### 2. 开始游戏

1. 在首页选择想玩的游戏（狼人杀 / 谁是卧底 / 阿瓦隆）
2. 输入你的游戏名字
3. 选择游戏人数
4. 为每个 AI 玩家分配想用的模型
5. 点击 **「开始游戏」**，享受对战！

### 3. 游戏中操作

- **发言**：在输入框中输入文字后按回车或点击发送按钮
- **投票**：在投票阶段通过按钮选择目标
- **特殊操作**：根据你的角色，在夜晚完成对应的操作（如预言家查验、女巫用药）

## 🏗️ 技术架构

```
AI-game-center/
├── index.html              # 入口 HTML
├── vite.config.js          # Vite 配置 + LLM 代理插件
├── package.json            # 项目配置
├── public/                 # 静态资源
│   ├── favicon.svg         # 网站图标
│   └── icons.svg           # 图标集
└── src/
    ├── main.js             # 应用入口
    ├── app.js              # 主应用控制器（路由、导航、设置）
    ├── ai-service.js       # AI 服务层（多档案管理、API 调用）
    ├── game-registry.js    # 游戏注册表（自动发现与注册）
    ├── style.css           # 全局样式
    └── games/
        ├── base-game.js    # 游戏基类（公共 UI 与逻辑）
        ├── werewolf.js     # 狼人杀游戏逻辑（警长版）
        ├── undercover.js   # 谁是卧底游戏逻辑
        ├── avalon.js       # 阿瓦隆游戏逻辑
        └── TEMPLATE.js     # 新游戏开发模板
```

### 关键设计

- **Vite 代理插件**：通过自定义 Vite 插件 (`llmProxyPlugin`) 在服务端转发 API 请求，避免浏览器 CORS 限制
- **多档案系统**：支持同时管理多个 AI 模型配置，存储在 `localStorage` 中
- **游戏注册表**：基于 `registerGame()` 的自动注册机制，添加新游戏无需修改主应用代码
- **游戏基类**：`BaseGame` 抽象了公共 UI 渲染、玩家交互和 AI 调用逻辑
- **独立记忆**：每个 AI 玩家维护独立的 `privateMemory`，包含角色信息和历史行动
- **上下文构建**：根据游戏状态、公开记录和私有记忆为每次 AI 调用构建完整的 context
- **静默操作**：非当前玩家阵营的夜间操作以静默模式进行，不向用户泄露信息

## 📝 开发相关

### 开发命令

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动开发服务器（含热更新） |
| `npm run build` | 构建生产版本 |
| `npm run preview` | 预览生产构建 |

### 添加新游戏

1. 复制 `src/games/TEMPLATE.js` 并重命名
2. 实现游戏类（继承 `BaseGame`）
3. 在文件末尾调用 `registerGame()` 注册
4. 在 `src/main.js` 中导入该文件即可，无需修改其他代码
